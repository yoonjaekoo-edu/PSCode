# 릴리즈 노트 (Release Notes)

## [2026-06-07] 마크다운 뷰어 개선 (LaTeX 수식 및 상세 헤더 지원)

### 변경 사항
- **LaTeX 수학 수식 렌더링 지원 (KaTeX 도입)**:
  - 마크다운 내 인라인 수식(`\( ... \)` 및 `$ ... $`)과 블록 수식(`\[ ... \]` 및 `$$ ... $$`)을 미려하게 렌더링하도록 KaTeX 라이브러리를 추가하고 스타일을 연동하였습니다.
- **모든 마크다운 헤더 수준 지원**:
  - 기존에 `###`까지만 파싱되던 제한을 제거하고, `#`부터 `######`까지 모든 수준(1~6단계)의 마크다운 헤더를 정상적으로 인식하여 적절한 크기와 가독성 높은 디자인으로 렌더링하도록 개선하였습니다.

## [2026-05-25] MD 파일 분할 화면, 문제 MD 생성 버튼, 에디터 언어 자동 감지

### 변경 사항
- **MD 파일 분할 화면 지원**:
  - `.cpp` 파일 열람 시 같은 이름의 `.md` 파일이 있으면 좌우 분할 화면(Split View)으로 함께 표시 (좌측: 코드 에디터, 우측: 마크다운 뷰어).
  - 분할 화면의 구분선을 드래그하여 좌우 비율을 자유롭게 조절 가능.
  - 매칭되는 `.md` 파일이 없으면 기존처럼 단일 에디터 화면으로 동작.
- **문제 MD 만들기 버튼 추가**:
  - 사이드바(확장/축소 상태 모두)에 "문제 MD 만들기" 버튼 추가.
  - 현재 열려 있는 `.cpp` 파일과 같은 이름의 `.md` 파일을 생성하고 분할 화면으로 즉시 열어줌.
  - 버튼은 `.cpp` 파일이 열려 있을 때만 활성화됨.
- **MD 파일 편집 가능하도록 개선**:
  - `.md` 파일을 클릭하면 마크다운 뷰어 대신 Monaco 에디터에서 직접 편집 가능.
  - 에디터 언어를 파일 확장자에 따라 자동 감지 (`.cpp` → C++, `.md` → Markdown).
- **분할 화면 레이아웃 버그 수정**:
  - 분할 화면에서 Monaco 에디터가 정상적으로 렌더링되지 않던 문제 수정.

## [2026-05-23] VSCode 스타일 파일 트리, C++ STL 스니펫 뷰어 및 포맷터 추가

### 변경 사항
- **VSCode 스타일 파일 탐색기 및 파일 관리자 구현**:
  - 사이드바 내에 디렉토리/파일 트리를 재귀적으로 조회하는 `WorkspaceTree` 컴포넌트 추가.
  - 마우스 우클릭을 통해 작동하는 인터랙티브 컨텍스트 메뉴 제공 (새 파일 생성, 새 폴더 생성, 이름 바꾸기, 삭제 연동).
  - 마우스 호버 시 바로 동작하는 퀵 버튼(새 파일/새 폴더/이름 바꾸기) 탑재.
  - 디렉토리 및 파일 수정 사항이 실제 디스크에 즉시 반영되며, 활성 파일 이름 변경/삭제 시 에디터 상태도 자동으로 동기화됩니다.
  - 기존 "오늘의 파일" 및 "최근 파일" 섹션을 화면 하단으로 축소하고 접기/펴기(Collapsible) 레이아웃으로 개선하여 작업 영역을 효율적으로 활용할 수 있게 하였습니다.
- **C++ STL 스니펫 라이브러리 모달 추가**:
  - C++ STL 스니펫 리스트를 한눈에 검색하고 조망할 수 있는 미려한 `SnippetsModal` 기획 및 탑재.
  - 사이드바(확장/축소 상태 모두 지원)에 "STL 스니펫 목록" 실행 단축 버튼 추가.
  - 선택한 스니펫의 코드를 바로 클립보드로 복사하는 기능 및 에디터의 현재 커서 위치에 Monaco의 템플릿 플레이스홀더를 유지한 채 인터랙티브하게 삽입하는 기능 추가.
- **사용자 정의 C++ 자동 정렬(Auto Formatter) 엔진 구축**:
  - `int a=1+b+c`와 같은 대입 및 수식을 `int a = 1 + b + c`로 공백 정렬하는 고성능 토크나이저 기반 C++ 포맷터 자체 설계 및 내장.
  - `Alt+Shift+F` 단축키, 상태바의 "코드 정렬" 버튼 및 커맨드 팔레트와 완벽히 유기적으로 작동.
  - 주석(`//`, `/* */`), 문자열 리터럴, 프리프로세서(#) 라인은 물론, 템플릿 괄호(예: `vector<int>`) 및 단항 증감 연산자(`i++`) 등 C++의 복잡한 신택스를 정밀 탐지하여 원본 코드를 파괴하지 않고 안전하게 정렬해 줍니다.

## [2026-05-19] 빌드 및 배포 시스템 개선

### 변경 사항
- **사용자 정의 커밋 메시지**: `build-deploy.ps1`에서 대화형으로 커밋 메시지를 입력받을 수 있도록 업데이트했습니다.
- **경고 메시지 억제**: `$WarningPreference` 설정 및 `npm`, `git`의 quiet 플래그를 사용하여 `build-deploy.ps1` 실행 시 콘솔 출력을 깔끔하게 정리했습니다.
- **프로젝트 규칙 수립**: 배포 워크플로우와 릴리즈 노트 작성 규칙을 담은 `GEMINI.md`를 생성했습니다. (한국어 작성 원칙 적용)
